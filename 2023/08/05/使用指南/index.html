2、新建文章

使用命令来新建文章

hexo new "title"

例如 hexo new "使用指南"

之后会生成一个 “使用指南.md”的文档：

在生成路径下即可找到生成的markdown文档，点击进入此文档即可开始编辑文档。

编辑文档

为文档添加分类以及标签，在date一行的下面，添加categories和tags属性（注意空格）：

之后在横线的下方写入文章的内容即可。

生成/删除静态文件及部署

博客编写结束之后即可预览文章效果以及部署到博客上。

删除静态文件：hexo c

生成静态文件：hexo g

启用本地端口：hexo s

​ 正常情况下，每次启动本地端口前只需更新静态文件即可。如果发生了一些错误，可以先删除静态文件，然后再重新生成。

​ 生成静态文件之后，便要启动本地端口。然后访问命令行返回的地址便能在本地访问到博客了。此时博客还未上传，可以达到预览效果。

重新部署：hexo d

 将修改后的内容同步到github中，从而更新博客内容。

总结博客编写完成之后的步骤为：

  • 删除已有静态文件：hexo clean 简写 hexo c
  • 重新生成静态文件：hexo g
  • 启动本地端口查看效果：hexo s (查看结束ctrl+c结束本地端口)
  • 重新部署：hexo d，稍等一段时间后刷新网页，发现主题生效
